## 🎯 核心思路

### 传统方式 vs 新方式对比

#### ❌ 传统方式：手动拖拽布局
```
UI设计稿 → Qt Designer拖拽 → 手动调整坐标 → 反复调试
```
**问题：**
- 效率低：一个界面需要1-2小时
- 精度差：手动拖拽难以精确对齐
- 难维护：修改UI需要重新拖拽
- 不可复用：每个界面都要重新做

#### ✅ 新方式：AI辅助代码生成
```
UI设计稿 → 整理坐标 → AI生成代码 → 一键运行
```
**优势：**
- 效率高：一个界面只需10-15分钟（**提升6倍**）
- 精度高：完全按照设计稿像素级还原
- 易维护：修改坐标即可重新生成
- 可复用：建立标准化流程，适用所有项目

## 🔄 完整工作流程

### 阶段1：UI设计对接（详见文档02）

```
1. 获取UI设计稿（PSD/Sketch/Figma）
2. 确认基准分辨率（如1920×1080、1024×768）
3. 获取切图资源（PNG/JPG）
4. 获取设计标注（坐标、尺寸、颜色、字体）
```

**关键点：**
- 要求UI设计师提供**图层分离**的设计稿
- 每个元素单独导出为PNG（背景、按钮、图标等）
- 提供完整的坐标标注（X、Y、Width、Height）

---

### 阶段2：坐标整理（详见文档03）

使用标准化表格整理所有UI元素：

| 序号 | 图层名称 | 元素类型 | 文件名 | X | Y | Width | Height | 父元素 | 备注 |
|-----|---------|---------|--------|---|---|-------|--------|--------|------|
| 1 | 背景 | 图片 | bg.png | 0 | 0 | 1024 | 768 | - | 全屏背景 |
| 2 | 侧边栏 | 图片 | sidebar.png | 17 | 79 | 122 | 675 | - | 左侧导航 |
| 3 | 监控按钮 | 按钮 | btn_jiankong.png | 28 | 106 | 105 | 90 | - | 有悬停态 |

**工具：**
- Excel/WPS表格
- HTML可视化模板（本项目提供）
- CSV格式（便于导入导出）

---

### 阶段3：AI生成代码（详见文档04）

将整理好的坐标表格发给AI，提示词：

```
请根据以下UI布局规范生成Qt代码：

[粘贴表格内容]

要求：
- 使用Qt Widgets
- 绝对定位（setGeometry）
- 支持窗口缩放
- 按钮支持正常态/悬停态切换
```

**AI会生成：**
- MainWindow.h（头文件）
- MainWindow.cpp（实现文件）
- 完整的按钮事件槽函数
- 窗口缩放适配代码

---

### 阶段4：编译运行

```bash
cd project_dir
qmake
make
./release/app.exe
```

**验证：**
- ✅ 所有图片正确加载
- ✅ 坐标位置准确
- ✅ 按钮悬停效果正常
- ✅ 窗口缩放等比例适配

## 📊 效率对比

### 实际项目数据

**项目：国能机器人巡检系统**
- UI元素数量：17个（背景、侧边栏、6个按钮、2个信息框、8个标签）
- 基准分辨率：1024×768

| 方式 | 时间 | 精度 | 可维护性 |
|-----|------|------|---------|
| 手动拖拽 | 90分钟 | 中等（±5px误差） | 差（修改需重新拖拽） |
| AI生成 | 15分钟 | 高（像素级精确） | 优（修改坐标即可） |

**效率提升：6倍**
