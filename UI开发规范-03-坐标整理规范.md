## 🎯 为什么需要坐标整理

### 问题场景

**设计师给的原始资料：**
```
- 一堆PNG图片
- 一个PSD设计稿
- 口头说明："按钮在左边，大概这个位置"
```

**开发者面临的困境：**
- ❌ 不知道每个元素的精确坐标
- ❌ 不知道图层叠加顺序
- ❌ 不知道哪些是按钮，哪些是静态图片
- ❌ 反复测量、调试、返工

---

### 解决方案：标准化坐标整理

**整理后的效果：**
```
✅ 每个元素的X、Y、Width、Height清晰明确
✅ 图层顺序（从下到上）一目了然
✅ 元素类型（图片/按钮/输入框）明确标注
✅ 可以直接交给AI生成代码
```

**效率对比：**
| 方式 | 时间 | 错误率 |
|-----|------|--------|
| 边开发边测量 | 60分钟 | 高（需反复调整） |
| 先整理后开发 | 10分钟整理 + 5分钟开发 | 低（一次到位） |

---

## 📊 坐标整理标准格式

### 核心字段说明

#### 1. 序号（必填）
- **作用：** 控制图层叠加顺序
- **规则：** 从1开始，数字越大越在上层
- **示例：** 1（背景）、2（侧边栏）、3（按钮）

#### 2. 图层名称（必填）
- **作用：** 描述元素的功能
- **规则：** 简洁明了，使用中文或英文
- **示例：** 背景、侧边栏、监控按钮、公司logo

#### 3. 元素类型（必填）
- **作用：** 区分静态图片和交互元素
- **可选值：**
  - `图片` - 静态显示，不可点击
  - `按钮` - 可点击，有事件响应
  - `输入框` - 可输入文字
  - `标签` - 显示文字（动态更新）
  - `QML组件` - 复杂的动态组件

#### 4. 文件名（必填）
- **作用：** 对应的切图文件名
- **规则：** 与实际文件名完全一致（包括扩展名）
- **示例：** bg.png、btn_login.png、sidebar.png

#### 5. X坐标（必填）
- **作用：** 元素左上角距离窗口左边的像素距离
- **单位：** 像素（px）
- **示例：** 0、100、256

#### 6. Y坐标（必填）
- **作用：** 元素左上角距离窗口顶部的像素距离
- **单位：** 像素（px）
- **示例：** 0、50、128

#### 7. Width宽度（必填）
- **作用：** 元素的宽度
- **单位：** 像素（px）
- **示例：** 1920、200、120

#### 8. Height高度（必填）
- **作用：** 元素的高度
- **单位：** 像素（px）
- **示例：** 1080、600、40

#### 9. 父元素（必填）
- **作用：** 标注元素的嵌套关系
- **规则：**
  - 填 `-` 表示窗口的直接子元素（推荐）
  - 填父元素名称表示嵌套关系
- **示例：** `-`、背景、侧边栏

#### 10. 备注（选填）
- **作用：** 补充说明
- **内容：**
  - 按钮状态（有悬停态、有按下态）
  - 特殊要求（需要动态更新、需要透明背景）
  - 文件名问题（文件名不一致、缺少文件）

---

### 标准表格格式

```
| 序号 | 图层名称 | 元素类型 | 文件名 | X | Y | Width | Height | 父元素 | 备注 |
|-----|---------|---------|--------|---|---|-------|--------|--------|------|
```

**示例：**

| 序号 | 图层名称 | 元素类型 | 文件名 | X | Y | Width | Height | 父元素 | 备注 |
|-----|---------|---------|--------|---|---|-------|--------|--------|------|
| 1 | 背景 | 图片 | bg.png | 0 | 0 | 1024 | 768 | - | 全屏背景 |
| 2 | 公司logo | 图片 | logo.png | 20 | 20 | 150 | 50 | - | 左上角 |
| 3 | 侧边栏 | 图片 | sidebar.png | 0 | 80 | 200 | 688 | - | 左侧导航栏 |
| 4 | 监控按钮 | 按钮 | btn_monitor.png | 20 | 100 | 160 | 50 | - | 有悬停态btn_monitor_hover.png |
| 5 | 主面板 | 图片 | panel.png | 220 | 100 | 784 | 648 | - | 右侧内容区 |

## 🔗 图层关系处理

### 两种布局方式

#### 方式A：平铺式（推荐）⭐

**特点：**
- 所有元素都是窗口的直接子元素
- 父元素都填 `-`
- 坐标都是相对于窗口左上角的绝对坐标

**优点：**
- ✅ 简单直观，易于理解
- ✅ 坐标清晰，不会混淆
- ✅ 移动元素不影响其他元素
- ✅ AI容易理解和生成代码

**示例：**

```
窗口 (1024×768)
├─ 背景图 (0, 0, 1024, 768)
├─ 侧边栏 (0, 80, 200, 688)
├─ 按钮1 (20, 100, 160, 50)
├─ 按钮2 (20, 170, 160, 50)
└─ 主面板 (220, 100, 784, 648)
```

**表格填写：**

| 序号 | 图层名称 | X | Y | 父元素 | 备注 |
|-----|---------|---|---|--------|------|
| 1 | 背景 | 0 | 0 | - | 最底层 |
| 2 | 侧边栏 | 0 | 80 | - | 视觉上在背景上 |
| 3 | 按钮1 | 20 | 100 | - | 视觉上在侧边栏上 |
| 4 | 按钮2 | 20 | 170 | - | 视觉上在侧边栏上 |
| 5 | 主面板 | 220 | 100 | - | 视觉上在背景上 |

---

#### 方式B：嵌套式（不推荐）

**特点：**
- 元素有父子关系
- 子元素的坐标是相对于父元素的
- 父元素移动时，子元素跟随移动

**缺点：**
- ❌ 坐标计算复杂（需要累加父元素坐标）
- ❌ 移动父元素会影响所有子元素
- ❌ AI理解困难，容易出错

**示例：**

```
窗口 (1024×768)
├─ 背景图 (0, 0, 1024, 768)
└─ 侧边栏 (0, 80, 200, 688)
    ├─ 按钮1 (20, 20, 160, 50)  ← 相对于侧边栏
    └─ 按钮2 (20, 90, 160, 50)  ← 相对于侧边栏
```

**表格填写：**

| 序号 | 图层名称 | X | Y | 父元素 | 备注 |
|-----|---------|---|---|--------|------|
| 1 | 背景 | 0 | 0 | - | 窗口子元素 |
| 2 | 侧边栏 | 0 | 80 | - | 窗口子元素 |
| 3 | 按钮1 | 20 | 20 | 侧边栏 | 相对侧边栏坐标 |
| 4 | 按钮2 | 20 | 90 | 侧边栏 | 相对侧边栏坐标 |

**注意：** 按钮1的绝对坐标是 (0+20, 80+20) = (20, 100)

---

### 如何选择布局方式？

**推荐使用平铺式，除非：**
- 设计师明确要求嵌套关系
- 元素需要跟随父元素移动（如弹窗内的按钮）
- 有大量重复的子元素（如列表项）

**90%的情况下，使用平铺式即可！**

## 🖥️ 使用HTML模板整理

### 模板文件

本项目提供了可视化HTML模板：
- 文件名：`UI布局规范模板.html`
- 位置：项目根目录

### 使用步骤

#### 1. 打开模板

```bash
# 双击打开，或在浏览器中打开
UI布局规范模板.html
```

#### 2. 填写项目信息

在"项目基本信息"部分填写：
- 项目名称：如"国能机器人巡检系统"
- 基准分辨率：如"1024 × 768"
- 缩放比例：如"100% (1.0)"

#### 3. 填写图层列表

在"图层列表"表格中，点击单元格直接编辑：

**示例填写：**
```
序号: 1
图层名称: 背景
元素类型: 图片
文件名: bg.png
X: 0
Y: 0
Width: 1024
Height: 768
父元素: -
备注: 全屏背景
```

#### 4. 填写交互元素

在"交互元素"表格中，填写所有按钮、输入框等：

**示例填写：**
```
序号: 1
元素名称: 监控按钮
类型: button
X: 28
Y: 106
Width: 105
Height: 90
父元素: -
文字: （留空，使用图片）
图片: btn_jiankong.png / btn_jiankong1.png
事件: 切换到监控页面
```

#### 5. 填写特殊要求

在"特殊要求"部分，补充说明：
```
要求1: 窗口固定大小1024×768
要求2: 按钮有正常态和悬停态
要求3: 图片从images/目录加载
```

#### 6. 导出数据

**方法1：全选复制**
```
1. Ctrl+A（全选）
2. Ctrl+C（复制）
3. 粘贴到文本文件或直接发给AI
```

**方法2：保存HTML**
```
1. Ctrl+S（保存）
2. 保存为新文件名（如：项目名_UI坐标.html）
3. 随时打开编辑
```

## ⚠️ 常见错误与避免方法

### 错误1：坐标单位混淆

**错误示例：**
```
X: 100mm
Y: 50cm
Width: 10%
```

**正确做法：**
```
X: 100        (单位：像素px，不需要写单位)
Y: 50
Width: 200
```

**说明：** 所有坐标和尺寸都是像素（px），不要使用其他单位！

---

### 错误2：相对坐标和绝对坐标混淆

**错误示例：**
```
侧边栏: X=0, Y=80
按钮: X=20, Y=20  (以为是相对侧边栏)
```

**实际结果：**
- 按钮显示在 (20, 20)，而不是 (20, 100)
- 按钮跑到窗口左上角了

**正确做法（平铺式）：**
```
侧边栏: X=0, Y=80, 父元素=-
按钮: X=20, Y=100, 父元素=-  (绝对坐标)
```

---

### 错误3：序号顺序错误

**错误示例：**
```
序号1: 按钮（最上层）
序号2: 背景（最下层）
```

**实际结果：**
- 背景显示在按钮上面
- 按钮被遮挡，看不见

**正确做法：**
```
序号1: 背景（最下层）
序号2: 按钮（最上层）
```

**记忆技巧：** 序号 = 图层顺序，从下到上递增

---

### 错误4：文件名不匹配

**错误示例：**
```
表格中填写: btn_login.png
实际文件名: btn_Login.png  (大小写不一致)
```

**实际结果：**
- 图片加载失败
- 显示空白

**正确做法：**
```
1. 统一使用小写文件名
2. 填写表格时，复制粘贴文件名（避免手打错误）
3. 填写完成后，检查文件是否存在
```

---

### 错误5：忘记填写按钮悬停态

**错误示例：**
```
文件名: btn_login.png
备注: （空）
```

**实际结果：**
- 只有正常态，没有悬停效果
- 用户体验差

**正确做法：**
```
文件名: btn_login.png
备注: 有悬停态btn_login_hover.png
```

或者在"交互元素"表格中：
```
图片: btn_login.png / btn_login_hover.png
```

### 案例对比：手动开发 vs 规范化流程

#### 手动开发（传统方式）

```
1. 打开Qt Creator
2. 拖拽一个QLabel
3. 打开设计稿，测量坐标
4. 手动输入X、Y、Width、Height
5. 设置图片路径
6. 编译运行，发现位置不对
7. 返回第3步，重新测量
8. 重复10次...
9. 添加按钮点击事件
10. 添加悬停效果
11. 调试、调试、调试...

总耗时：90分钟
```

#### 规范化流程（本方法）

```
1. 打开HTML模板
2. 填写所有元素坐标（10分钟）
3. 复制粘贴给AI
4. AI生成代码（1分钟）
5. 编译运行（2分钟）
6. 完成！

总耗时：15分钟
```

**效率提升：6倍**
