## 🎨 UI设计基础概念

### 1. 图层（Layer）

**什么是图层？**
- 设计软件（PS/Sketch/Figma）中的独立元素
- 类似透明胶片叠加在一起
- 每个图层可以独立移动、编辑、导出

**图层顺序与Qt代码的关系：**
```
设计软件中：
├─ 图层1（最上层）  →  Qt代码中：最后创建
├─ 图层2
├─ 图层3
└─ 图层4（最下层）  →  Qt代码中：最先创建
```

**示例：**
```cpp
// 设计软件图层顺序（从下到上）：
// 1. 背景图
// 2. 侧边栏
// 3. 按钮

// Qt代码创建顺序（从下到上）：
lblBackground = new QLabel(this);  // 序号1，最底层
lblSidebar = new QLabel(this);     // 序号2，在背景上
btnMenu = new QLabel(this);        // 序号3，在侧边栏上
```

### 2. 切图（Slicing）

**什么是切图？**
- 将设计稿中的元素导出为独立的图片文件
- 每个可见元素都需要单独导出

**切图规范：**
```
✅ 正确的切图：
images/
├─ bg.png              (背景图)
├─ sidebar.png         (侧边栏)
├─ btn_menu.png        (按钮-正常态)
├─ btn_menu_hover.png  (按钮-悬停态)
└─ logo.png            (公司logo)

❌ 错误的切图：
images/
└─ all_in_one.png      (所有元素合并在一张图)
```

**切图命名规范：**
- 使用英文小写 + 下划线
- 描述性命名：`btn_login.png`（登录按钮）
- 状态后缀：`_normal`、`_hover`、`_pressed`、`_disabled`

### 3. 坐标系统

**设计稿坐标系：**
```
(0,0) ────────────► X轴
  │
  │    [元素]
  │    (x, y, width, height)
  │
  ▼
 Y轴
```

**坐标标注示例：**
```
按钮位置：
- X: 100px（距离左边100像素）
- Y: 200px（距离顶部200像素）
- Width: 120px（宽度120像素）
- Height: 40px（高度40像素）
```

---

### 4. 基准分辨率

**什么是基准分辨率？**
- 设计稿的画布尺寸
- 所有坐标都基于这个尺寸标注

**常见分辨率：**
```
桌面应用：
- 1920×1080（全高清）
- 1366×768（笔记本常见）
- 1024×768（工业触摸屏）

移动应用：
- 375×667（iPhone 8）
- 414×896（iPhone 11）
```

**重要：** 必须明确基准分辨率，否则坐标会错位！

## 📞 与UI设计师的沟通清单

### 第一次对接（项目启动）

**需要确认的信息：**

#### 1. 基准分辨率
```
问题：这个界面的设计分辨率是多少？
示例回答：1024×768
```

#### 2. 图层结构
```
问题：能否提供图层分离的设计稿？
要求：每个元素独立图层，不要合并
```

#### 3. 切图格式
```
问题：切图导出为PNG还是JPG？
建议：PNG（支持透明背景）
```

#### 4. 按钮状态
```
问题：按钮有几种状态？
常见状态：
- 正常态（normal）
- 悬停态（hover）
- 按下态（pressed）
- 禁用态（disabled）
```

#### 5. 字体和颜色
```
问题：文字使用什么字体？颜色值是多少？
示例：
- 字体：微软雅黑 14px
- 颜色：#333333
```

### 设计稿交付时（开发前）

**检查清单：**

#### ✅ 必须提供
- [ ] 完整的设计稿源文件（PSD/Sketch/Figma）
- [ ] 所有切图资源（PNG格式）
- [ ] 坐标标注文档（Excel/PDF）
- [ ] 字体文件（如果使用特殊字体）

#### ✅ 切图资源检查
- [ ] 文件命名规范（英文小写+下划线）
- [ ] 尺寸正确（与标注一致）
- [ ] 背景透明（PNG格式）
- [ ] 按钮状态齐全（正常态+悬停态）

#### ✅ 坐标标注检查
- [ ] 包含所有元素的X、Y、Width、Height
- [ ] 坐标单位为像素（px）
- [ ] 基准分辨率明确标注

## ❓ 常见问题与解决方案

### 问题1：设计师只给了一张完整的效果图

**问题描述：**
- 所有元素合并在一张图里
- 无法单独导出每个元素

**解决方案：**
```
方案A：要求重新切图（推荐）
- 礼貌地说明需要图层分离的设计稿
- 提供切图规范文档

方案B：自己切图（不推荐）
- 使用PS/Figma打开设计稿
- 手动选择区域导出
- 容易出错，效率低
```

---

### 问题2：坐标标注不完整

**问题描述：**
- 只标注了部分元素
- 缺少Width/Height

**解决方案：**
```
方案A：使用设计软件测量
1. 打开设计稿源文件
2. 选中元素，查看属性面板
3. 记录X、Y、Width、Height

方案B：使用标注工具
- 蓝湖（lanhuapp.com）
- 摹客（mockplus.cn）
- 自动生成标注
```

---

### 问题3：按钮只有一种状态

**问题描述：**
- 设计师只提供了正常态
- 缺少悬停态、按下态

**解决方案：**
```
方案A：要求补充（推荐）
- 说明需要悬停态（鼠标移上去的效果）
- 通常是改变颜色或添加阴影

方案B：程序实现（备选）
- 使用QSS样式表实现悬停效果
- 代码示例：
  QPushButton:hover {
      background-color: #0078d7;
  }
```

---

### 问题4：分辨率不匹配

**问题描述：**
- 设计稿是1920×1080
- 实际运行设备是1366×768

**解决方案：**
```
方案A：等比例缩放（推荐）
- 使用本规范的缩放适配方案
- 详见文档05：缩放适配方案

方案B：重新设计
- 要求设计师提供多套设计稿
- 成本高，不推荐
```

---

### 问题5：文件命名混乱

**问题描述：**
- 文件名是中文：`按钮1.png`
- 文件名无意义：`image001.png`

**解决方案：**
```
方案A：要求重命名（推荐）
- 提供命名规范文档
- 示例：btn_login.png、icon_user.png

方案B：自己重命名（备选）
- 批量重命名工具
- 注意更新坐标清单中的文件名
```

## 🎓 学习建议

### 初学者

1. **了解设计软件基础**
   - 学习Figma基础操作（免费在线工具）
   - 了解图层、切图、标注的概念

2. **实践练习**
   - 下载免费设计稿
   - 自己尝试切图和标注
   - 理解设计师的工作流程

3. **建立沟通习惯**
   - 主动询问不清楚的地方
   - 使用标准化的沟通模板
   - 记录常见问题和解决方案

---

### 进阶者

1. **学习设计规范**
   - Material Design（Google）
   - Human Interface Guidelines（Apple）
   - 了解设计原则和最佳实践

2. **建立资源库**
   - 收集常用的UI组件
   - 整理切图资源
   - 建立团队共享库

3. **优化协作流程**
   - 使用协作工具（蓝湖、摹客）
   - 建立自动化标注流程
   - 减少重复沟通
