## 🔄 从坐标到代码的转换规则

### 基本转换规则

#### 规则1：图片元素 → QLabel

**坐标表格：**
```
序号: 1
图层名称: 背景
元素类型: 图片
文件名: bg.png
X: 0, Y: 0, Width: 1024, Height: 768
```

**生成代码：**
```cpp
// 头文件 mainwindow.h
QLabel *lblBackground;

// 实现文件 mainwindow.cpp
lblBackground = new QLabel(this);
lblBackground->setGeometry(0, 0, 1024, 768);
lblBackground->setPixmap(QPixmap(imagePath + "bg.png"));
lblBackground->setScaledContents(true);
```

---

#### 规则2：按钮元素 → ClickableLabel

**坐标表格：**
```
序号: 4
图层名称: 监控按钮
元素类型: 按钮
文件名: btn_jiankong.png
X: 28, Y: 106, Width: 105, Height: 90
备注: 有悬停态btn_jiankong1.png
```

**生成代码：**
```cpp
// 头文件 mainwindow.h

```

#### 规则3：命名规范

**图层名称 → 变量名转换：**

| 图层名称 | 元素类型 | 变量名 | 规则 |
|---------|---------|--------|------|
| 背景 | 图片 | lblBackground | lbl + 英文名 |
| 侧边栏 | 图片 | lblSidebar | lbl + 英文名 |
| 监控按钮 | 按钮 | btnJiankong | btn + 拼音/英文 |
| 公司logo | 图片 | lblGsmc | lbl + 拼音缩写 |
| 运行状态标签 | 图片 | lblYunxingzhuangtai | lbl + 拼音 |

**命名前缀规则：**
- `lbl` - QLabel（图片、标签）
- `btn` - ClickableLabel（按钮）
- `txt` - QLineEdit（输入框）
- `cmb` - QComboBox（下拉框）
- `chk` - QCheckBox（复选框）

---

#### 规则4：序号 → 创建顺序

**坐标表格：**
```
序号1: 背景
序号2: 侧边栏
序号3: 按钮
```

**生成代码顺序：**
```cpp
// 按序号从小到大创建
lblBackground = new QLabel(this);  // 序号1
lblSidebar = new QLabel(this);     // 序号2
btnMenu = new ClickableLabel(this); // 序号3
```

**原因：** 后创建的控件显示在上层

## 🎨 ClickableLabel自定义组件

### 为什么需要ClickableLabel？

**问题：** Qt的QLabel不支持点击事件和悬停效果

**解决方案：** 自定义ClickableLabel类

### 功能特性

1. **点击事件**
   - 鼠标左键点击时发出 `clicked()` 信号
   - 可以连接到槽函数处理业务逻辑

2. **悬停效果**
   - 鼠标移入时显示悬停态图片
   - 鼠标移出时恢复正常态图片
   - 自动切换，无需手动控制

3. **鼠标指针**
   - 自动设置为手型指针（Qt::PointingHandCursor）
   - 提示用户可以点击

---

### 完整代码

```cpp
// ===== 头文件 =====
class ClickableLabel : public QLabel
{
    Q_OBJECT
public:
    explicit ClickableLabel(QWidget *parent = nullptr);
    
    // 设置正常态图片
    void setNormalImage(const QString &path);
    
    // 设置悬停态图片
    void setHoverImage(const QString &path);

signals:
    // 点击信号
    void clicked();

protected:
    // 鼠标按下事件
    void mousePressEvent(QMouseEvent *event) override;
    
    // 鼠标移入事件
    void enterEvent(QEvent *event) override;
    
    // 鼠标移出事件
    void leaveEvent(QEvent *event) override;

private:
    QString normalImagePath;  // 正常态图片路径
    QString hoverImagePath;   // 悬停态图片路径
};

// ===== 实现文件 =====
ClickableLabel::ClickableLabel(QWidget *parent)
    : QLabel(parent)
{
    // 设置鼠标指针为手型
    setCursor(Qt::PointingHandCursor);
}

void ClickableLabel::setNormalImage(const QString &path)
{
    normalImagePath = path;
    setPixmap(QPixmap(path));  // 默认显示正常态
}

void ClickableLabel::setHoverImage(const QString &path)
{
    hoverImagePath = path;
}

void ClickableLabel::mousePressEvent(QMouseEvent *event)
{
    if (event->button() == Qt::LeftButton) {
        emit clicked();  // 发出点击信号
    }
    QLabel::mousePressEvent(event);
}

void ClickableLabel::enterEvent(QEvent *event)
{
    // 鼠标移入，显示悬停态
    if (!hoverImagePath.isEmpty()) {
        setPixmap(QPixmap(hoverImagePath));
    }
    QLabel::enterEvent(event);
}

void ClickableLabel::leaveEvent(QEvent *event)
{
    // 鼠标移出，恢复正常态
    if (!normalImagePath.isEmpty()) {
        setPixmap(QPixmap(normalImagePath));
    }
    QLabel::leaveEvent(event);
}
```

## 📂 图片路径处理

### 路径策略

#### 策略1：相对于exe的路径（推荐）⭐

```cpp
imagePath = QCoreApplication::applicationDirPath() + "/";
```

**目录结构：**
```
项目目录/
├─ build-xxx/
│   └─ debug/
│       ├─ app.exe
│       ├─ bg.png
│       ├─ sidebar.png
│       └─ btn_login.png
```

**优点：**
- ✅ 简单直接
- ✅ 图片和exe在同一目录
- ✅ 便于调试

**缺点：**
- ❌ 图片和exe混在一起，不够整洁

---

#### 策略2：images子目录（推荐）⭐⭐

```cpp
imagePath = QCoreApplication::applicationDirPath() + "/images/";
```

**目录结构：**
```
项目目录/
├─ build-xxx/
│   └─ debug/
│       ├─ app.exe
│       └─ images/
│           ├─ bg.png
│           ├─ sidebar.png
│           └─ btn_login.png
```

**优点：**
- ✅ 图片集中管理
- ✅ 目录结构清晰
- ✅ 便于维护

**缺点：**
- ⚠️ 需要手动创建images目录并拷贝图片

---

#### 策略3：Qt资源文件（不推荐）

```cpp
imagePath = ":/images/";
```

**优点：**
- ✅ 图片编译到exe中
- ✅ 不需要单独拷贝图片

**缺点：**
- ❌ 修改图片需要重新编译
- ❌ exe文件变大
- ❌ 不适合频繁更换UI

---
