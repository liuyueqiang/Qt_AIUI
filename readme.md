# Qt UI快速开发指南 - AI辅助生成界面

## 🎯 核心思路

**传统方式：** UI设计稿 → Qt Designer拖拽 → 手动调整 → 反复调试（90分钟）

**新方式：** UI设计稿 → 整理坐标 → AI生成代码 → 一键运行（15分钟）

**效率提升：6倍**

---

## 📋 完整流程（3步）

### 第1步：获取UI资源

从UI设计师那里获取：

1. **切图文件**（PNG格式）
   ```
   bg.png              - 背景图
   sidebar.png         - 侧边栏
   btn_login.png       - 按钮正常态
   btn_login_hover.png - 按钮悬停态
   logo.png            - 公司logo
   ```

2. **基准分辨率**
   ```
   例如：1920×1080 或 1024×768
   ```

3. **坐标信息**
   ```
   每个元素的 X、Y、Width、Height
   ```

---

### 第2步：整理坐标表格

使用提供的 `UI布局规范模板.html`，填写表格：

| 序号 | 图层名称 | 元素类型 | 文件名 | X | Y | Width | Height | 父元素 | 备注 |
|-----|---------|---------|--------|---|---|-------|--------|--------|------|
| 1 | 背景 | 图片 | bg.png | 0 | 0 | 1024 | 768 | - | 全屏背景 |
| 2 | 侧边栏 | 图片 | sidebar.png | 0 | 80 | 200 | 688 | - | 左侧导航 |
| 3 | 登录按钮 | 按钮 | btn_login.png | 20 | 100 | 160 | 50 | - | 有悬停态btn_login_hover.png |

**关键点：**
- **序号**：从1开始，数字越大越在上层（后创建的显示在上面）
- **元素类型**：图片（静态）或 按钮（可点击）
- **父元素**：都填 `-`（表示平铺式布局，简单直观）
- **备注**：标注按钮的悬停态文件名

---

### 第3步：AI生成代码

**操作：**
1. 打开 `UI布局规范模板.html`
2. Ctrl+A（全选）
3. Ctrl+C（复制）
4. 粘贴给AI

**提示词：**
```
请根据以下UI布局规范生成Qt代码：

[粘贴HTML内容]

要求：
- 使用Qt Widgets
- 绝对定位（setGeometry）
- 支持窗口缩放
- 按钮支持正常态/悬停态切换
- 生成完整的MainWindow.h和MainWindow.cpp
```

**AI会生成：**
- mainwindow.h（头文件）
- mainwindow.cpp（实现文件）
- 包含所有UI元素
- 包含按钮点击事件
- 包含窗口缩放代码

## 📐 图层顺序规则

**设计软件中的图层顺序 = Qt代码创建顺序**

```
设计软件（从下到上）：     Qt代码（从上到下）：
┌─────────────┐           lblBackground = new QLabel(this);  // 序号1
│   按钮      │  ←序号3    lblSidebar = new QLabel(this);     // 序号2
├─────────────┤           btnLogin = new QLabel(this);       // 序号3
│   侧边栏    │  ←序号2
├─────────────┤
│   背景图    │  ←序号1
└─────────────┘
```

**记忆技巧：** 序号小的先创建，显示在底层

## 🎨 图片命名规范

### 基本规范

```
✅ 正确命名：
bg.png                  - 背景
sidebar.png             - 侧边栏
btn_login.png           - 登录按钮
btn_login_hover.png     - 登录按钮悬停态
icon_user.png           - 用户图标
logo_company.png        - 公司logo

❌ 错误命名：
背景.png                - 不要用中文
image001.png            - 无意义命名
Button1.png             - 大小写不统一
```

### 命名规则

1. **使用英文小写 + 下划线**
   ```
   btn_login.png  ✅
   btnLogin.png   ❌
   Btn-Login.png  ❌
   ```

2. **描述性命名**
   ```
   btn_login.png      - 登录按钮
   btn_register.png   - 注册按钮
   icon_settings.png  - 设置图标
   ```

3. **状态后缀**
   ```
   btn_login.png         - 正常态
   btn_login_hover.png   - 悬停态
   btn_login_pressed.png - 按下态
   btn_login_disabled.png- 禁用态
   ```

4. **常用前缀**
   ```
   bg_     - 背景（background）
   btn_    - 按钮（button）
   icon_   - 图标（icon）
   logo_   - 标志（logo）
   panel_  - 面板（panel）
   label_  - 标签（label）
   ```

## 🔧 坐标整理技巧

### 如何获取坐标？

#### 方法1：使用设计软件

**Figma：**
```
1. 选中元素
2. 右侧属性面板查看：
   X: 100
   Y: 200
   W: 120
   H: 40
```

**Photoshop：**
```
1. 选中图层
2. 窗口 → 信息面板
3. 查看 X、Y、W、H
```

#### 方法2：使用标注工具

**蓝湖（lanhuapp.com）：**
```
1. 上传设计稿
2. 自动生成标注
3. 点击元素查看坐标
```

**摹客（mockplus.cn）：**
```
1. 导入设计稿
2. 自动标注
3. 导出坐标数据
```

---

### 坐标单位

**重要：所有坐标都是像素（px）**

```
✅ 正确：
X: 100
Y: 200
Width: 120
Height: 40

❌ 错误：
X: 100px      - 不要写单位
Y: 10cm       - 不要用其他单位
Width: 50%    - 不要用百分比
```

## 🖼️ 图层关系：平铺式 vs 嵌套式

### 平铺式（推荐）⭐

**特点：** 所有元素都是窗口的直接子元素

```
窗口
├─ 背景图 (0, 0, 1024, 768)
├─ 侧边栏 (0, 80, 200, 688)
├─ 按钮1 (20, 100, 160, 50)
└─ 按钮2 (20, 170, 160, 50)
```

**表格填写：**
| 序号 | 图层名称 | X | Y | 父元素 |
|-----|---------|---|---|--------|
| 1 | 背景 | 0 | 0 | - |
| 2 | 侧边栏 | 0 | 80 | - |
| 3 | 按钮1 | 20 | 100 | - |
| 4 | 按钮2 | 20 | 170 | - |

**优点：**
- ✅ 简单直观
- ✅ 坐标都是绝对坐标
- ✅ AI容易理解

**90%的情况使用这种方式！**

---

### 嵌套式（不推荐）

**特点：** 元素有父子关系

```
窗口
├─ 背景图 (0, 0, 1024, 768)
└─ 侧边栏 (0, 80, 200, 688)
    ├─ 按钮1 (20, 20, 160, 50)  ← 相对侧边栏
    └─ 按钮2 (20, 90, 160, 50)  ← 相对侧边栏
```

**表格填写：**
| 序号 | 图层名称 | X | Y | 父元素 |
|-----|---------|---|---|--------|
| 1 | 背景 | 0 | 0 | - |
| 2 | 侧边栏 | 0 | 80 | - |
| 3 | 按钮1 | 20 | 20 | 侧边栏 |
| 4 | 按钮2 | 20 | 90 | 侧边栏 |

**缺点：**
- ❌ 坐标计算复杂
- ❌ AI容易出错

## 🚀 窗口缩放适配

### 为什么需要缩放？

```
设计稿：1920×1080
实际屏幕：1366×768  ← 显示不全！

解决方案：等比例缩放
```

### 实现原理

```cpp
// 1. 计算缩放比例
scaleX = 实际宽度 / 基准宽度
scaleY = 实际高度 / 基准高度
scale = min(scaleX, scaleY)  // 取较小值，保持比例

// 2. 更新元素位置
实际X = 基准X × scale
实际Y = 基准Y × scale
实际Width = 基准Width × scale
实际Height = 基准Height × scale
```

### 代码示例

```cpp
// 头文件
void resizeEvent(QResizeEvent *event) override;
void updateLayout();
double scale;

// 实现文件
void MainWindow::resizeEvent(QResizeEvent *event)
{
    QMainWindow::resizeEvent(event);
    updateLayout();
}

void MainWindow::updateLayout()
{
    // 计算缩放比例
    double scaleX = (double)width() / 1024.0;
    double scaleY = (double)height() / 768.0;
    scale = qMin(scaleX, scaleY);

    // 更新所有元素
    lblBackground->setGeometry(0, 0, 1024 * scale, 768 * scale);
    lblSidebar->setGeometry(0, 80 * scale, 200 * scale, 688 * scale);
    btnLogin->setGeometry(20 * scale, 100 * scale, 160 * scale, 50 * scale);
}
```

## ⚠️ 常见错误

### 错误1：序号顺序错误

```
❌ 错误：
序号1: 按钮（最上层）
序号2: 背景（最下层）

结果：背景遮挡按钮

✅ 正确：
序号1: 背景（最下层）
序号2: 按钮（最上层）
```

---

### 错误2：文件名不匹配

```
❌ 错误：
表格中：btn_login.png
实际文件：btn_Login.png  ← 大小写不一致

结果：图片加载失败

✅ 正确：
统一使用小写文件名
```

---

### 错误3：忘记标注悬停态

```
❌ 错误：
文件名: btn_login.png
备注: （空）

结果：按钮没有悬停效果

✅ 正确：
文件名: btn_login.png
备注: 有悬停态btn_login_hover.png
```
